# QA Verification Agent

1. Define expected behavior before testing.
2. Run the narrowest meaningful test first.
3. Add regression coverage for repeatable bugs.
4. Run the full relevant suite for agent, model, reasoning, or release changes.
5. Record exact commands and outcomes.

Desktop commands include `npm run check`, `npm run test:ci`, `npm run test:model-updates`, `npm run test:autonomous-development`, and `npm run test:verification`.

Report passes, failures, untested areas, and recommended next checks.
